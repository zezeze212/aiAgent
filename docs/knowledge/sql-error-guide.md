# SQL 报错排查知识库

## Unknown column

### 典型报错

`java.sql.SQLSyntaxErrorException: Unknown column 'xxx' in 'field list'`

这个错误表示：SQL 中引用了某个字段，但数据库当前表结构中不存在这个字段。

### 常见原因

- Mapper XML 中写了数据库不存在的字段；
- 注解 SQL 中使用了旧字段名；
- MyBatis-Plus 的 QueryWrapper / LambdaQueryWrapper 使用了错误字段；
- 实体类字段和数据库字段映射不一致；
- 数据库没有执行最新 DDL；
- 本地数据库和测试/生产数据库表结构不一致；
- 查询字段来自旧版本代码，但表结构已经调整。

### 排查顺序

1. 从报错信息中提取缺失字段名；
2. 确认报错 SQL 涉及的表名；
3. 查询数据库真实表结构；
4. 判断字段是否真的不存在；
5. 检查 Mapper XML、注解 SQL、QueryWrapper；
6. 检查实体类字段和 `@TableField` 映射；
7. 检查是否漏执行 DDL；
8. 如果字段已废弃，应修改 SQL 或代码逻辑。

### 修复建议

如果字段确实不存在，优先确认业务逻辑应该使用哪个字段。

如果是代码写错字段，应修改 Mapper XML、注解 SQL 或 QueryWrapper。

如果是数据库缺字段，应补充 DDL，并确认不同环境表结构一致。

如果是实体映射问题，应检查实体类字段名、数据库字段名和 MyBatis-Plus 映射规则。

### Agent 回答要求

当工具已经查询到真实表结构时，回答必须优先基于工具结果，不要凭空猜测字段是否存在。

回答中应该包含：

- 缺失字段名；
- 涉及表名；
- 真实表结构中是否存在该字段；
- 可能出问题的代码位置；
- 建议排查和修复顺序。
